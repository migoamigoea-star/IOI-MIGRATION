# Project Overview

## Introduction

The IOI Migration Project is an initiative to migrate existing business operations and data to **Microsoft Dynamics 365 Business Central**. The project involves custom AL extension development, data migration, integration configuration, and user training.

## Objectives

- **Modernize** the current business platform by adopting Dynamics 365 Business Central
- **Migrate** historical data with full validation and reconciliation
- **Extend** the standard Business Central functionality with custom AL extensions tailored to IOI's specific requirements
- **Integrate** with existing third-party systems where applicable
- **Train** end users and administrators on the new platform

## Scope

### In Scope

- Analysis and mapping of existing business processes
- Custom AL extension development
- Master data and transactional data migration
- User acceptance testing (UAT)
- Go-live support and hypercare period

### Out of Scope

- Changes to non-migrating legacy systems
- Third-party integrations not explicitly defined in the project requirements

## Stakeholders

| Role | Responsibility |
|---|---|
| Project Sponsor | Executive oversight and final approvals |
| Project Manager | Day-to-day project coordination and reporting |
| Business Analyst | Requirements gathering and process mapping |
| AL Developer | Custom extension development and testing |
| Data Migration Lead | Data extraction, transformation, and loading |
| End Users | UAT participation and feedback |

## Key Milestones

| Milestone | Description |
|---|---|
| Project Kick-off | Initial planning and stakeholder alignment |
| Requirements Complete | All business requirements documented and signed off |
| Development Complete | All AL extensions developed and unit tested |
| UAT Complete | User acceptance testing passed |
| Go-Live | Production deployment and hypercare begins |
| Project Close | Post-go-live review and documentation handover |
