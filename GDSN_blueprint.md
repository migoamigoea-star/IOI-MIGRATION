## Purpose

GDSN records all gift receipts, donations made, and sponsorships given or received by IOI Acidchem
staff, ensuring compliance with the company's anti-bribery and anti-corruption policy. Each
notification is submitted by the relevant employee and approved by their HOD. The migrated solution
must preserve item-level declaration details, SAP reference linkage, total value computation, and
HOD attribution for audit evidence.

## Migration Risks & Notes

1. **TotalValue computation:** Domino computes Total = Quantity × UnitPrice via formula field. In
   canvas v3, `TotalValue` must be computed in Power Automate on submit and stored — never rely on
   canvas formula for persistent computed values. Display formula in screen only; stored value from
   PA.

2. **SAP linkage (SAP_ID / PONumber):** The form has SAP reference fields. These are free-text in
   Domino. In M365, maintain as free-text — do not build SAP live lookup unless explicitly scoped.
   Validate format via `IsMatch()` if a known SAP ID pattern is defined.

3. **Anti-bribery audit trail:** All GDSN records must be immutable post-HOD decision
   (`IsLocked=Yes`). Any correction requires a new GDSN record. Enforce via `DisplayMode` locked on
   all edit screens once `IsLocked=Yes`.

4. **Anonymous visibility concern:** Rejected records should not be visible to other employees.
   Implement gallery filter: `CreatedBy = CurrentUser()` OR user is in `D05-HR-Manager` group.

5. **PO conditional visibility:** PONumber field should only be visible/editable when `IsPO = Yes`.
   Implement `Visible` property on PONumber control conditioned on `IsPO` toggle value.
