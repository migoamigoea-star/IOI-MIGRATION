## Purpose

PAF records the annual performance appraisal cycle for Executive-grade and above staff at IOI
Acidchem. The form captures employee details, dual-appraiser scoring (direct supervisor and
second-level), HOD commentary, Division Head sign-off, and a numeric rating on a 55–100 scale. The
migrated solution must preserve dual-appraiser attribution, per-category rating history via child
records, and a locked final state that prevents post-approval score edits.

## Workflow Stage Map

```
[HR / Initiator] → STAGE 1: Draft
      ↓ Submit
[Appraiser 1] → STAGE 2: Appraiser 1 Review (rate + comment)
      ↓ Submit scores
[Appraiser 2] → STAGE 3: Appraiser 2 Review (optional 2nd level)
      ↓ Submit scores
[HOD] → STAGE 4: HOD Review (comment + acknowledge)
      ↓ Submit
[Division Head] → STAGE 5: Division Head Approval (final sign-off)
      ↓ Approve / Reject
STAGE 6: Approved (IsLocked = Yes)
```

| Stage | Action                 | Actor Role      | SP Group             | Power Automate Trigger                                           |
| ----- | ---------------------- | --------------- | -------------------- | ---------------------------------------------------------------- |
| 1     | Create & Submit        | HR or Initiator | D05-HR-Staff         | When Status='Draft' and item created                             |
| 2     | Appraiser 1 scores     | Appraiser 1     | D05-HR-Appraisers    | When Status='Draft' → submitted → notify Appraiser1              |
| 3     | Appraiser 2 scores     | Appraiser 2     | D05-HR-Appraisers    | When Status='Appraiser1_Review' → App1 submits → notify App2     |
| 4     | HOD review             | HOD             | D05-HR-HOD           | When Status='Appraiser2_Review' → notify HOD                     |
| 5     | Division Head approval | Division Head   | D05-HR-DivisionHeads | When Status='HOD_Review' → notify DivHead                        |
| 6     | Close / Lock           | System          | —                    | When Status='DivHead_Review' and DivHead approves → IsLocked=Yes |

## Migration Risks & Notes

1. **Dual appraiser score aggregation:** Domino computes composite scores from per-category entries.
   In M365, composite scores must be calculated in Power Automate (iterate child
   `HR_PAF_RatingDetails`, sum weighted scores) and written back to parent `OverallScore`. Risk:
   race conditions if both appraisers submit simultaneously — mitigate with sequential workflow
   stages.

2. **Child table rating categories:** The 11-page PDF implies multiple rating categories. These must
   be pre-seeded as rows in `HR_PAF_RatingDetails` when the form is created, then updated in place
   by each appraiser. Craftsman must implement an EditForm over the child gallery, not flat columns.

3. **Optional Appraiser 2:** `AppBy2` may be optional for some employee grades. The workflow must
   skip Stage 3 if Appraiser2 is not assigned. Power Automate condition: if `Appraiser2` is null
   after App1 submits, advance directly to Stage 4 (HOD).

4. **Score locking after approval:** Once `IsLocked=Yes`, no edits to scores or comments must be
   possible. Implement `IsLocked` check on all Edit screen buttons and form `DisplayMode`.

5. **Annual cycle management:** Multiple PAF records per employee (one per year) require that the
   gallery default-filters to the current `AppraisalYear`. HR manager must be able to view
   historical years. Implement year-picker filter in `PAF_List`.
