# Blueprint — Talent Management - Planning & Update (TMPU)

---

## 1. Purpose

The **Talent Management - Planning & Update (TMPU)** form is IOI's structured talent pipeline
management tool. HR selects high-potential employees, assigns a PIC/HOD to conduct a readiness
analysis, and together they build an Individual Development Plan (IDP) aligned to the IOI Leadership
Competency Framework. The completed assessment drives promotion, succession, and training investment
decisions.

This form migrates to Microsoft 365 as a Power Apps canvas app backed by `MainDB_HR` (FormType =
"TMPU"), replacing the Lotus Domino Talent Management module.

## 4. Workflow

**Complexity:** 3-stage sequential (no formal approval gate — HR internal process)

### Stage Flow Diagram

```
[Stage 1: Initiation — HR Business Partner]
         │ submit
         ▼
[Stage 2: Readiness Analysis + IDP — PIC / HOD]
         │ submit assessment
         ▼
[Stage 3: Endorsement + Finalisation — HR Admin / HOD]
         │ endorse
         ▼
[Finalised]
```

**CurrentStatus transitions:**

```
Initiation → Nomination/Readiness → IDP Review → Finalised
```

### Stage Matrix

| Stage | Name                       | Trigger                        | Actor                    | Actions                                                         | Next Stage | Notifications                         |
| ----- | -------------------------- | ------------------------------ | ------------------------ | --------------------------------------------------------------- | ---------- | ------------------------------------- |
| 1     | Initiation                 | New TMPU item created          | HR Business Partner      | Select employee, fill profile, assign PIC/HOD, submit           | 2          | PIC / HOD notified to begin assessment|
| 2     | Readiness Analysis + IDP   | Status=Nomination/Readiness    | PIC / HOD (Editor1)      | Complete strengths, competency ratings, IDP activities, submit  | 3          | HR Admin notified of completed assessment |
| 3     | Endorsement + Finalisation | Status=IDP Review              | HR Admin / HOD (Editor2) | Review completed IDP, endorse, mark Finalised                   | Finalised  | Employee and HR BP notified of outcome |

### Trigger-Condition Matrix

| Stage            | Trigger Condition              | Required Checks                              | Advance Path                      | Return/Reject Path                      |
| ---------------- | ------------------------------ | -------------------------------------------- | --------------------------------- | --------------------------------------- |
| Initiation       | Item created with FormType=TMPU| FullName, EmpNo, PIC, DOB, Dept present      | Route to PIC for assessment       | Incomplete form returned to HR BP       |
| Readiness / IDP  | Status=Nomination/Readiness    | All competency rows rated; IDP has 1+ entry  | Submit for HR Admin endorsement   | HR Admin returns with feedback to PIC   |
| Endorsement      | Status=IDP Review              | Strengths, competencies, IDP all complete    | Set Status=Finalised              | Return to PIC for revision              |

### Power Automate Flows

| Flow Name                  | Trigger                            | Key Actions                                                                          |
| -------------------------- | ---------------------------------- | ------------------------------------------------------------------------------------ |
| `TMPU_OnInitiate`          | Item created, FormType=TMPU        | Generate INO (`HR-TMP-YYMM-NNNN`); set Status=Initiation; stamp DateSubmitted; notify PIC |
| `TMPU_OnReadinessSubmit`   | Status updated to IDP Review       | Stamp ReadinessSubmittedDate; notify HR Admin of completed assessment                |
| `TMPU_OnEndorse`           | Status updated to Finalised        | Set CurrentStatus=Finalised; stamp EndorsedDate; notify Employee and HR BP           |

## 6. Key Migration Notes

1. **Repeating rows normalised:** Domino flat columns (S1/S2/S3 + 6 competency rows) must be
   migrated to child SharePoint lists (`HR_TMPU_Strengths`, `HR_TMPU_Competencies`, `HR_TMPU_IDP`).
   Migration script must translate flat rows to child records.

2. **Gallery/repeating section UI:** The Assessment screen requires gallery-within-form or a
   repeating section pattern for Strengths, Competencies, and IDP. Use Power Apps `Gallery` inside a
   `VerticalContainer` for each section.

3. **Computed fields:** Age (DOB → years), Year (DateComm → years), must be computed client-side in
   canvas. Do not attempt to store derived values — compute on-the-fly from base dates.

4. **INO sequential numbering:** `HR-TMP-YYMM-NNNN` — the PA flow must handle concurrent submissions
   gracefully (use SP item ID suffix as tie-breaker if needed).

5. **Photo attachment:** Employee photo stored as attachment or SP image column. Keep as attachment
   to avoid SP image column limitations on bulk records.

6. **DEC-001 compliance:** FormType = "TMPU" discriminator allows live submission into shared
   `MainDB_HR` list.

7. **DEC-004 compliance:** EnvironmentTag (DEV/TEST/PROD) on all records. Each environment must be
   isolated in canvas filter.
