## Purpose

EEN manages the formal exit lifecycle for employees leaving IOI Acidchem — covering resignation,
termination, retirement, and end-of-contract scenarios. The form collects exit details at
initiation, routes through HOD and Division Head approvals, and closes with HR recording the actual
last working day, system deactivation date, and exit interview outcome. The migrated solution must
preserve attributable decision history, exit interview tracking, and date accuracy for payroll and
IT system deactivation.

## Workflow Stage Map

```
[Employee/HR] → STAGE 1: Draft
      ↓ Submit
[HOD] → STAGE 2: HOD Review
      ↓ Approve / Reject
[Division Head] → STAGE 3: Division Head Review
      ↓ Approve / Reject
[HR] → STAGE 4: HR Processing (set ActualLWD + dtDeactivate + HRComment)
      ↓ Close
STAGE 5: Closed (IsLocked = Yes)
```

## Migration Risks & Notes

1. **Dual date tracking (LastWorkingDay vs ActualLWD):** Domino stores both requested and confirmed
   last working days. Power Apps must enforce that ActualLWD is only editable by HR at Stage 4, not
   by the initiator. Risk: accidental override in Edit screen — mitigate with `IsLocked` and
   role-based visibility.

2. **IT deactivation dependency (dtDeactivate):** The Domino form generates an IT action via
   separate workflow. In M365, a Power Automate flow on `dtDeactivate` set must email IT (or create
   a Planner task). This integration is outside Power Apps canvas scope — document as external
   dependency.

3. **Exit interview tracking:** `ExitInterview` flag (Yes/No) triggers a reminder in Domino. In
   M365, a scheduled reminder flow should fire if `ExitInterview=Yes` and no `EDDate` is recorded
   within N days of `LastWorkingDay`. Implement as Power Automate scheduled flow.

4. **Multi-site records (PRAI + JOHOR):** The `Site` column must be populated accurately for correct
   SharePoint list routing. A single `MainDB_HR` list covers both sites — filter by Site in gallery
   views.

5. **Termination vs resignation visibility:** Employee-initiated resignations should allow the
   employee to view their own record. Terminations should restrict employee visibility to approved
   HR disclosure only. Implement using `ExitCategory` column + row-level security in Power Apps
   (filter gallery by `CurrentUser()` ownership).
